TRIGGERS

  • Hovering on the "Sign In" button.
  • Hovering off the "Sign In" button.
  • Clicking on the "Sign In" button with the correct sign-in information.
  • Clicking on the "Sign In" button with the incorrect sign-in information.
  • Clicking on the "Sign In" button while one/both of the input boxes are left empty.

RULES

HOVER ON/OFF SIGN IN BUTTON

  • ON: "Sign-In" button adopts a white background with black text and a black border.
  • ON: "Sign-In" button cursor changes to a hand pointer.
  • OFF: "Sign-In" button changes back to it's original black background with white text while keeping the black border.
  • OFF: "Sign-In" button cursor changes back to the standard pointer arrow.

CLICK ON SIGN IN BUTTON: CORRECT INFO

  • Sign-in information is being processed.
  • Information database is being updated.
  • Logo div on right side will fade to a GIF of a basketball being scored to signify a succesful sign-in.
  • Logo div GIF on right side will also contain text that says "SIGN IN SUCCESSFUL!".
  • Email and password paragraph text will turn dark green to signify a successful sign-in.
  • Input boxes will turn pale green to signify a successful sign-in.

CLICK ON SIGN IN BUTTON: INCORRECT INFO

  • Sign-in information is being processed.
  • Information database is being updated.
  • Logo div on right side will fade to a GIF of a basketball clanking off the rim to show an unsuccesful sign-in.
  • Logo div GIF on right side will also contain text that says "UNSUCESSFUL... TRY AGAIN".
  • Email and password paragraph text will turn dark red to signify an unsuccessful sign-in.
  • Input boxes will turn pale red to signify an unsuccessful sign-in.

CLICK ON SIGN IN BUTTON: EMPTY INPUT TEXT

  • Sign-in information is being processed.
  • Information database is being updated.
  • If any input box is left empty when the sign-in button gets clicked, a small GIF of a referee will fade in and display a "TIMEOUT!" message to signify the error.
  • Email and password paragraph text will turn dark red to signify the error.
  • Input boxes will turn pale red to signify the error.

LOOPS/MODES

LOOPS

  • None.

MODES

  • If the information is incorrect and the "Sign In" button is clicked, the unsuccessful GIF and red warnings are displayed. But then if the correct information is input and the button gets clicked again, the unsuccesful GIF will switch to the succesful GIF and the red warnings will switch to green (and vice versa).
  • If the input boxes are empty and the "Sign In" button is clicked, the ref warning GIF and red warnings will display. But then if the input boxes are filled in and the button gets clicked again, the ref warning GIF will dissapear. However, if the input info entered happens to be incorrect, then the red warnings will stay on screen and the unsuccesful GIF displays. If the input info happens to be correct, then the red warnings change to green and the succesful GIF displays.
