NBA ID Sign-In

Joe Chindemi - IDM-241-001 Microinteractions

TRIGGERS

  • Users can initiate the "Sign In" button with a click, which triggers a microinteraction.

RULES

  • If an email or password is not entered and the "Sign In" button is triggered, then a red requirement message will pop up below the email or password section asking for a valid input.
  • If an invalid email or password is entered and the "Sign In" button is triggered, then a red requirement message will pop up below the email or password section asking for a valid input.
  • If a valid email and password is entered and the "Sign In" button is triggered, a short loading window will overlay the screen. Then, the user is transfered to the homepage fully signed into their account.

FEEDBACK

  • When a user activley hovers on the "Sign In" button, the button changes from a black background with white text to a white background with black text and a black border. This signifys a potential trigger for the user to initiate.
  • When a user activley hovers on the "Sign In" button, the mouse changes to a clicker icon to signify an interactive element.
  • When email or password is incorrect or not entered and a user triggers the "Sign In" button, the red requirement message will pop up signifying an incorrect entry. This also encourages the user to try again with feedback on why their entry did not work.
  • When the email and password is correct and the user triggers the "Sign In" button, the user will be presented an overly loading screen that signifies that the microinteraction has been properly initiated. This loading screen consists of a 50% grey overlay with a quick loading wheel animation.

LOOPS/MODES

  • Once the user signs in to their account, there is no longer a sign in option available.
  • In order to access the sign in again, users must click the "Sign Out" button on the menu bar or in their settings.
  • NBA.com will remember the user's NBA ID upon revisiting the site, unless the user chooses to sign out.
