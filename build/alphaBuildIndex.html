TRIGGERS

  • To start the interaction, users initiate the "Sign In" button with a click.

RULES

  • The only way this interaction is triggered is when the user correctly inputs their information and clicks on the "Sign In" button.
  • NBA.com will remember the user's NBA ID upon revisiting the site, unless the user chooses to sign out.

FEEDBACK

  • When a user activley hovers on the "Sign In" button, the button changes from a black background with white text to a white background with black text and a black border. This signifys a potential trigger for the user to initiate.
  • When a user activley hovers on the "Sign In" button, the mouse changes to a clicker icon to signify an interactive element.
  • When a user clicks on the "Sign In" button and their information is valid, a short loading overlay will appear.
  • The low-opacity, grey overlay is displayed on the full screen with an animated loading circle. This sequence only lasts for a second before the sign-in is "initiated" and the overlay fades away.

LOOPS/MODES

  • Once the user signs in to their account, there is no longer a sign in option available.
  • In order to access the sign in again, users must click the "Sign Out" button on the menu bar or in their settings. After they have signed out, they can sign back in and retrigger the button loading sequence.
